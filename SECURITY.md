# Security policy

## Supported versions

Security fixes are applied to the latest minor release.

| Version | Supported |
| --- | --- |
| 1.2.x | Yes |
| Earlier | No |

## Report a vulnerability

Do not disclose a suspected vulnerability in a public issue. Use GitHub's
private vulnerability reporting page for this repository:

<https://github.com/shahriarfarzadi/run-ab-experiments-skill/security/advisories/new>

Include the affected version, harness, reproduction steps, impact, and any
suggested mitigation. You should receive an acknowledgment within seven days.

This policy covers the installer, packaged scripts, workflows, and repository
distribution. Methodological disagreements that do not create a software or
data-security risk belong in a normal issue.
